В коде НЕ указано, на каком сервере запущен бот. Это ты настраиваешь сам
(VPS, хостинг, другой ПК, где крутится python bot.py с твоим TELEGRAM_TOKEN).

ЧТО СДЕЛАТЬ:

1) Узнать, где запущен бот
   - Вспомни, где ты впервые запускал бота (этот ПК или арендованный сервер?).
   - Если на VPS: зайди по SSH на тот хост, где ставил бота.
   - Репозиторий в GitHub: https://github.com/gpro7724-netizen/russian-weather-tg-bot

2) Обновить код НА ТОМ СЕРВЕРЕ (или ПК), откуда бот реально отвечает
   Вариант А — через Git (если на сервере клонирован репозиторий):
     cd /путь/к/russian-weather-tg-bot
     git pull origin main
     # Остановить бота (Ctrl+C или systemctl stop ... или pm2 stop ...)
     python bot.py
     # или: nohup python bot.py &   /  systemctl start ...  /  pm2 start ...

   Вариант Б — скопировать файл вручную:
     Скопируй с ЭТОГО компьютера файл bot.py из папки
       c:\Users\Administrator\Desktop\tg bot2\russian-weather-tg-bot\bot.py
     на сервер в папку, где там лежит бот. Перезапусти бота на сервере.

3) Убедиться, что запущена новая версия
   - В Telegram запроси погоду по любому городу.
   - Если обновилось: будет строка "Местное время (GMT+..., по городу): ЧЧ:ММ, день, дата"
     и НЕ будет слова "UTC".
   - Если по-прежнему "19:21 UTC" — бот всё ещё запущен со старым bot.py
     (другой сервер или процесс не перезапустили).

4) Если бот запущен только на ЭТОМ ПК
   - Закрой все окна/терминалы с python.
   - Запусти: дважды щёлкни ЗАПУСТИТЬ_БОТА.bat в этой папке.
   - Проверь в Telegram.
